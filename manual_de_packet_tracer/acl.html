11. Listas de Control de Acceso (ACL)

Las Listas de Control de Acceso (ACL) son un conjunto de reglas que se utilizan para filtrar el tráfico de red en dispositivos como puede ser routers y switches. Estas reglas incrementan la seguridad de la red controlando el flujo de paquetes, permitiendo o denegando el tráfico bajo una serie de criterios específicos.

11.1 Ejemplos de Uso Común de ACL

  • Restricción de Acceso: Limitan el acceso a partes específicas de la red, un ejemplo puede ser en un colegio bloqueando casas de apuestas, o juegos o material pornografico.

  • Control de Tráfico: Administran el flujo de tráfico, pudiendo bloquear o permitir tipos específicos de tráfico como servicios de web (HTTP), conexiones remotas (SSH), entre otros.

  • Seguridad de Red: Ayudan a prevenir ataques de red y acceso no autorizado, siendo esenciales en la implementación de políticas de seguridad.

11.2 Tipos de ACL

  1. ACL Estándar:
    • Se enfocan en las direcciones IP de origen para permitir o denegar el tráfico.
    • Intervalo de identificación de ACL: 1-99 y 1300-1999.
  2. ACL Extendida:
    • Ofrecen un control más detallado, filtrando por direcciones IP de origen y destino, protocolos de red y números de puerto.
    • Intervalo de identificación de ACL: 100-199 y 2000-2699.

11.3 Configuración de una ACL Estándar

  1. Crear la ACL Estándar:
    • Sintaxis básica para crear una regla en la ACL:
Router(config)# access-list [número] [permit|deny] [dirección origen] [máscara wildcard]
  • Ejemplo: Para permitir el tráfico de la red 192.168.1.0:
access-list 10 permit 192.168.1.0 0.0.0.255
  1. Aplicar la ACL a una Interfaz:
    • Seleccionar la interfaz y aplicar la ACL:
Router(config)# interface [tipo de interfaz][número de interfaz]
  • Ejemplo de interfaz: 
Interface gigabitethernet 0/0/0
  • Aplicación en una interfaz:
Router(config-if)# ip access-group [número de ACL] [in|out]
  • Ejemplo: Para aplicar la ACL en la interfaz entrante:
ip access-group 10 in

11.4 Configuración de una ACL Extendida

  1. Crear la ACL Extendida:

    • Sintaxis para crear una regla en la ACL extendida: 
      Router(config)# access-list [número] [permit|deny] [protocolo] [IP origen] [wildcard origen] [IP destino] [wildcard destino] [operador] [puerto]
    • Ejemplo: Para permitir el tráfico HTTP de la red 192.168.1.0 a la red 192.168.2.0: 
      access-list 100 permit tcp 192.168.1.0 0.0.0.255 192.168.2.0 0.0.0.255 eq 80

  2. Aplicar la ACL a una Interfaz:

    • Seleccionar la interfaz y aplicar la ACL: 
      Router(config)# interface [tipo de interfaz][número de interfaz]
Router(config-if)# ip access-group [número de ACL] [in|out]
    • Ejemplo: Para aplicar la ACL en la interfaz saliente: 
      ip access-group 100 out

11.5 Ejemplos Avanzados de Configuración (Según el manual de Cisco Packet Tracer)

Ejemplo 1: ACL para Permitir Tráfico HTTPS y Denegar Tráfico HTTP

access-list 110 permit tcp any any eq 443
access-list 110 deny tcp any any eq 80
access-list 110 permit ip any any

Ejemplo 2: ACL para Bloquear Acceso a una Subred Específica

access-list 120 deny ip 192.168.10.0 0.0.0.255 any
access-list 120 permit ip any any

Ejemplo 3: ACL para Permitir Tráfico ICMP (Ping)

access-list 130 permit icmp any any echo-reply
access-list 130 permit icmp any any echo

Ejemplo 4: ACL para Bloquear Acceso a Puertos Específicos

access-list 140 deny tcp any any eq 23
access-list 140 deny udp any any eq 69
access-list 140 permit ip any any

11.6 Ejercicios de Juan Diego

Ejemplos Prácticos

Permitir una Red

Router0#configure terminal
Router0(config)#access-list 1 permit 192.168.1.0 0.0.0.255
Router0(config)#interface FA0/0
Router0(config-if)#ip access-group 1 out

Denegar un Host

Router0(config)#no access-list 1
Router0(config)#access-list 1 deny 192.168.1.10 0.0.0.0
Router0(config)#access-list 1 permit 192.168.1.0 0.0.0.255
Router0(config)#interface S0/0/0
Router0(config-if)#ip access-group 1 out

Prohibir FTP, pero Permitir Todo lo Demás

Router(config)#access-list 102 deny tcp 192.168.1.0 0.0.0.255 10.0.0.0 0.255.255.255 eq 21
Router(config)#access-list 102 permit ip any any
Router(config)#interface FA 0/1
Router(config-if)#ip access-group 102 in

R1(config)#access-list 104 permit tcp host 10.0.0.4 host 30.0.0.2 eq 23

Permitir Acceso Web desde 10.0.0.2 a 192.168.1.2

Router(config)#access-list 103 permit tcp host 10.0.0.2 host 192.168.1.2 eq 80
Router(config)#interface fastEthernet 0/0
Router(config-if)#ip access-group 103 in

Permitir Acceso Web desde Cualquier Máquina a 192.168.1.2

Router(config)#access-list 103 permit tcp any host 192.168.1.2 eq 80
Router(config)#interface fastEthernet 0/0
Router(config-if)#ip access-group 103 in

Permitir Acceso desde Cualquier Máquina a 192.168.1.3 en el Puerto 22, Denegar Acceso en otros Puertos

Router(config)#access-list 103 permit tcp any host 192.168.1.3 eq 22
Router(config)#access-list 103 deny tcp any host 192.168.1.3 neq 22
