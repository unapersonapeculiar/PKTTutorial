Manual de Configuración de NAT en Packet Tracer

Este manual describe los pasos para configurar NAT (Network Address Translation) en un router utilizando Cisco Packet Tracer.
La configuración permite que una red privada tenga acceso a una red pública y también habilita servidores en la red privada para ser accesibles desde la red pública.

Configuración de Interfaces

  1. Configurar la interfaz GigabitEthernet0/0/0 como interfaz externa:
interface GigabitEthernet0/0/0
ip address 10.0.0.1 255.255.255.0
ip nat outside
  1. Configurar la interfaz GigabitEthernet0/0/1 como interfaz interna:
interface GigabitEthernet0/0/1
ip address 192.168.200.1 255.255.255.0
ip nat inside

Configuración de NAT para Acceso de Red Privada a Red Pública

  1. Crear un pool de direcciones NAT: (Se crea el rango para las direcciones ip publicas del nat "SE UTILIZA la ip de la interfaz externa")
ip nat pool CONJUNTO1 10.0.0.1 10.0.0.1 netmask 0.0.0.255
  1. Configurar la lista de acceso para definir la red interna:
access-list 99 permit 192.168.200.0 0.0.0.255
  1. Asignar la lista de acceso al pool de NAT con sobrecarga:
ip nat inside source list 99 pool CONJUNTO1 overload

Configuración de NAT para Habilitar Servidores en la Red Privada (Abrir Puertos de toda la vida)

  1. Configurar NAT estático para permitir acceso a un servidor HTTP en la red privada:
ip nat inside source static tcp 192.168.200.100 80 10.0.0.1 80

  1. Configurar NAT estático para permitir acceso a un servidor FTP en la red privada:
ip nat inside source static tcp 192.168.200.100 21 10.0.0.1 21
  1. Configurar NAT estático para permitir acceso a un servidor DNS en la red privada:
ip nat inside source static udp 192.168.200.100 53 10.0.0.1 53

Notas Adicionales

  • Verificar la Configuración:
    Para verificar que la configuración de NAT está funcionando correctamente, puedes usar los siguientes comandos:
show ip nat translations
show ip nat statistics

  • Guardar la Configuración:
    No olvides guardar la configuración para que persista después de reiniciar el router:
end
write
reload

Aqui teneis para copiar y pegar los comandos en el router y hacerlo de una tacada. (Obviamente cambiar las ips para usar este documento en examen.)
